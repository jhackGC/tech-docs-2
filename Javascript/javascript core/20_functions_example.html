Functions are the building blocks of JavaScript! They're reusable pieces of code that perform specific tasks. Think of them as recipes - you write them once and can use them over and over with different ingredients (parameters).

üîç Function Anatomy

Parts of a Function:

function functionName (parameters) { code } return value;
function addTwo(number) { return number + 2; } const result = addTwo(5); // result is 7
  • function - Keyword that declares a function
  • addTwo - The function name (should be descriptive!)
  • (number) - Input parameters the function accepts
  • { } - Function body containing the code to execute
  • return - Sends a value back to whoever called the function

üéØ Different Types of Functions

Function Declaration

The classic way to define a function. Can be called before it's defined (hoisting).

// Function declaration function greet(name) { return `Hello, ${name}!`; } // Can be called before definition due to hoisting console.log(greet("Alice")); // "Hello, Alice!"

üéÆ Try Function Declaration:

Function Expression

Assign a function to a variable. Cannot be called before it's defined.

// Function expression const multiply = function(a, b) { return a * b; }; console.log(multiply(4, 5)); // 20

üßÆ Try Function Expression:

Arrow Function (ES6+)

Modern, concise syntax. Great for short functions and callbacks.

// Arrow function - full syntax const divide = (a, b) => { return a / b; }; // Arrow function - concise syntax (implicit return) const square = x => x * x; // Arrow function - no parameters const getRandomNumber = () => Math.random(); console.log(divide(10, 2)); // 5 console.log(square(4)); // 16 console.log(getRandomNumber()); // random number

üèπ Try Arrow Functions:

Anonymous Functions

Functions without names, often used as callbacks.

// Anonymous function as callback setTimeout(function() { console.log("This runs after 2 seconds"); }, 2000); // Anonymous arrow function as callback const numbers = [1, 2, 3, 4, 5]; const doubled = numbers.map(x => x * 2); console.log(doubled); // [2, 4, 6, 8, 10]

üß© Function Scope

Variables Inside and Outside Functions

let globalVariable = "I'm global!"; function demonstrateScope() { let localVariable = "I'm local!"; console.log(globalVariable); // Can access global console.log(localVariable); // Can access local return localVariable; } console.log(globalVariable); // Works fine // console.log(localVariable); // Error! Not accessible outside function

‚ö†Ô∏è Scope Rules:

  • Global variables: Accessible everywhere in your code
  • Local variables: Only accessible inside the function where they're defined
  • Parameters: Act like local variables inside the function
  • Avoid global variables: They can cause conflicts and bugs

üí° Best Practices

Function Best Practices:

  • Use descriptive names: calculateTax() instead of calc()
  • Keep functions small: One function should do one thing well
  • Use arrow functions for short operations: Great for callbacks and one-liners
  • Always return something meaningful: Even if it's just true or false
  • Validate parameters: Check if inputs are what you expect
  • Avoid global variables: Pass data through parameters instead
  • Document complex functions: Add comments explaining what they do

üéØ Common Use Cases for Functions:

  • Calculations: Math operations, conversions, validations
  • Data manipulation: Formatting, filtering, sorting arrays/objects
  • Event handling: Responding to user interactions
  • API calls: Fetching data from servers
  • Utilities: Helper functions for common tasks
  • Validation: Checking if data meets certain criteria
