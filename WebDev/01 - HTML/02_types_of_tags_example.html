HTML5 Semantic Tags

These tags give meaning to your content structure:

Page Structure Tags:

<header> - Page Header
<nav> - Navigation: Home About Contact
<main> - Main Content Area
<section> - Thematic Section

This represents a thematic grouping of content.

<article> - Self-contained Content

This could be a blog post, news article, or forum post.

<time> July 15, 2025
<aside> - Sidebar Content

Related links, advertisements, or tangential information.

<footer> - Page Footer

Text-Level Semantic Tags:

Here is some text with a <mark> highlighted section for emphasis.

Event date: <time> December 25, 2025

Interactive Content Tags:

<details> and <summary> - Click to expand

This content is hidden by default and only shows when you click the summary!

Perfect for FAQ sections or collapsible content.

<dialog> - This is a dialog box!

Progress and Meter Tags:

<progress> - Task completion:

70% 70% complete

<meter> - Measurement within a range:

Disk usage: 60% (60% full)

Score: 8 out of 10

Benefits of Semantic HTML5:

  • Accessibility: Screen readers understand content structure
  • SEO: Search engines better index semantic content
  • Maintainability: Code is self-documenting
  • Styling: CSS can target meaningful elements
  • Future-proofing: Works better with new technologies

Compare this to using generic <div> elements - semantic tags provide meaning!
